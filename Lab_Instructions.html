Lab 1: Implement Secure Data Communication with AES and RSA

In this lab, you will generate RSA keys, encrypt a symmetric key with the public key, and encrypt a file using AES.

Step 1: Generate RSA Keys

Generate 2048-bit RSA private and public keys using OpenSSL.

# Generate a 2048-bit RSA private key openssl genrsa -out private.pem 2048 # Derive the public key from the private key openssl rsa -in private.pem -pubout -out public.pem

Step 2: Encrypt a Symmetric Key with RSA

Generate a random 256-bit AES key and encrypt it with the RSA public key.

# Generate a random 256-bit AES key aes_key=$(openssl rand -base64 32) # Encrypt the AES key with the RSA public key encrypted_aes_key=$(openssl rsautl -encrypt -inkey public.pem -pubin -in $aes_key -out aes.key.enc)

Step 3: Encrypt a File with AES

Encrypt a sample file (plaintext.txt) using OpenSSL's AES-256-CBC algorithm.

# Create a sample plaintext file echo "Hello, World!" > plaintext.txt # Set the password for encryption (keep secret) aes_password="my_secret_password" # Encrypt the plaintext file with AES openssl enc -aes-256-cbc -salt -in plaintext.txt -out ciphertext.enc -pass file:$aes_password

Step 4: Decrypt the AES Key and File

Decrypt the AES key using the RSA private key, then decrypt the file.

# Decrypt the encrypted AES key with the RSA private key decrypted_aes_key=$(openssl rsautl -decrypt -inkey private.pem -pubin -in aes.key.enc -out aes.key.dec) # Decrypt the ciphertext file with the decrypted AES key openssl enc -aes-256-cbc -d -salt -in ciphertext.enc -out decrypted_ciphertext.txt -pass pass:$aes_password # Verify content matches diff plaintext.txt decrypted_ciphertext.txt

Lab 2: Configure Access Control Policies in a Database

Implement access control matrices (ACMs) and granular privileges in MySQL/PostgreSQL.

Step 1: Create a Database

CREATE DATABASE big_data_project; USE big_data_project;

Step 2: Define Roles and Users

Define three roles: Analyst, Data Engineer, and Admin.

-- Create roles CREATE ROLE analyst; CREATE ROLE data_engineer; CREATE ROLE admin; -- Create users with assigned roles CREATE USER 'analyst_user'@'%' IDENTIFIED BY 'password'; GRANT ANALYST privileges TO 'analyst_user'@'%'; CREATE USER 'data_engineer_user'@'%' IDENTIFIED BY 'password'; GRANT DATA_ENGINEER privileges TO 'data_engineer_user'@'%';

Step 3: Grant Privileges

Assign SELECT/INSERT/UPDATE permissions to specific tables.

-- Grant SELECT on patient_records for Analysts GRANT SELECT ON patient_records TO 'analyst_user'; -- Grant INSERT, UPDATE, DELETE on transaction_logs for Data Engineers GRANT INSERT, UPDATE, DELETE ON transaction_logs TO 'data_engineer_user';

Step 4: Audit Logging

Enable audit logging to monitor access.

SET GLOBAL log_output = 'TABLE'; -- Check logs SELECT * FROM mysql.general_log;

Lab 4: Case Study - Capital One Data Breach

Analyze the 2019 breach and design a response plan.

Incident Overview

  • Target: Capital One (2019)
  • Impact: 100M+ customers exposed.
  • Vulnerabilities: SSRF (Server Side Request Forgery), Misconfigured WAF.

Step 1: Analyze Causes

  • Technical: Unpatched vulnerabilities, Misconfigured firewalls.
  • Organizational: Inadequate access controls, Insufficient monitoring.

Step 2: Response Plan

Design an effective response plan using FTC guidelines.

  • Secure Systems: Immediately patch vulnerabilities and close access.
  • Forensics: Engage experts to identify root cause.
  • Notification: Inform stakeholders honestly without revealing sensitive info.

Step 3: Python Response Class

class ResponsePlan: def __init__(self): self.severity = None self.action = None def update(self, severity, action): self.severity = severity self.action = action print(f"Plan Updated: {severity} - {action}") breach = ResponsePlan() breach.update("Severe", "Notify affected users and patch WAF")

Lab 6: Differential Privacy

Implement Laplace Noise to protect data queries.

Step 1: Define Query

Target Query: Average Score of students.

Step 2: Implement Laplace Noise

Add noise drawn from a Laplace distribution scaled to sensitivity.

import numpy as np from scipy import stats def laplace_noise(true_answer, epsilon, delta): # Calculate scale (sigma) sigma = np.sqrt(2 * epsilon / delta) # Add noise noise = stats.laplace.rvs(loc=0, scale=sigma) return true_answer + noise # Example Usage true_avg = 85.5 epsilon = 1.0 # Privacy budget noisy_avg = laplace_noise(true_avg, epsilon, 1.0) print(f"True: {true_avg}, Noisy: {noisy_avg}")

Step 3: Evaluate Accuracy

Observe that as Epsilon increases (less privacy), accuracy improves (error decreases).
